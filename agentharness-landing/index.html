What it is

AgentHarness is a brief-to-PR pipeline. You write a short brief. A chain of specialised agents takes it from rough idea to a reviewable pull request — handoffs, plans, code, and review included — without you holding their hand.

  1. brief Your input
  2. analyst Scopes the work
  3. architect Shapes the system
  4. designer Designs the surface
  5. planner Breaks down the build
  6. developer Writes the code
  7. reviewer Audits and refines
  8. done Pull request open

Why use it

For developers who would rather ship than supervise.

Async by default

Submit a brief and go offline. Agents do not need you in the loop.

Fully autonomous

Every stage — analysis, design, planning, build, review — is handled end to end.

No babysitting

No standups, no nudges, no prompt-tuning. The pipeline runs itself.

Ships PRs

The output is a real pull request, not a transcript or a plan.

How it works

Four steps. The middle two are not your problem.

  1. Write the brief

    One paragraph. What you want and why. Plain language is fine.

  2. Submit it

    Drop the brief into AgentHarness and confirm. That is your last action.

  3. Walk away

    Agents run analyse, design, plan, build, review. Sleep, ship something else, take a walk.

  4. Review the PR

    The pull request is already open when you come back. Read the diff, merge, done.
